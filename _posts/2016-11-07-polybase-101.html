PolyBase is a technology that accesses and combines both non-relational and relational data, all from within SQL Server. It allows you to run queries on external data in Hadoop or Azure blob storage. The queries are optimized to push computation to Hadoop.

Polybase has been built to address four major usecases -
a) To allow corporates to reuse their current Data Analysts with SQL skills to access data on Hadoop using T-SQL
b) To provide capability for analysis to be be done on distributed datasets split between SQL Server and Hadoop without moving data around
c) To let all the current ETL/Reporting tools leverage native OLEDB drivers to connect to datasets on Hadoop.
d) To build Hybrid Data warehouse wherein all cold data is archived away to Hadoop whilst still being accessible for querying thereby reducing the cost of maintaining larger datasets.

Polybase achieves this by providing capability to create external table structures on top of data stored in HDFS. Thus, it provides capability to write T-SQL statements on these external tables and joining them with other tables (external/conventional) to write all forms of queries.
